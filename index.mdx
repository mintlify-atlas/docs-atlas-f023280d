`@deessejs/fp` brings functional programming primitives to TypeScript — without the learning curve of heavy FP libraries. Handle errors explicitly using `Result`, represent optional values with `Maybe`, wrap throwing code with `Try`, and chain async operations with `AsyncResult`. Every type integrates seamlessly with TypeScript's type system, giving you compile-time guarantees that errors are always handled.

## Why @deessejs/fp?

TypeScript's type system is powerful — but traditional error handling with `try/catch` punches holes in it. When a function can throw, callers have no idea. When `null` can appear, runtime crashes follow.

`@deessejs/fp` makes errors and optional values **part of the type**, so TypeScript can enforce they're always handled.

## Quick look

```typescript
import { ok, err, isOk, Result } from '@deessejs/fp'

class NotFoundError extends Error {
  constructor(public id: number) {
    super(`User ${id} not found`)
    this.name = 'NotFoundError'
  }
}
class UnauthorizedError extends Error {
  constructor() { super('Unauthorized'); this.name = 'UnauthorizedError' }
}

type UserError = NotFoundError | UnauthorizedError

interface User { id: number; name: string }

function getUser(id: number): Result<User, UserError> {
  if (id <= 0) return err(new NotFoundError(id))
  return ok({ id, name: 'Alice' })
}

const result = getUser(1)

if (isOk(result)) {
  console.log(result.value.name) // TypeScript knows this is User
} else {
  console.error(result.error.message) // TypeScript knows this is UserError
}
```
